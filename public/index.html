BÀI TẬP 7

Demo CI/CD Pipeline với GitLab Pages

Website tĩnh minh họa quy trình Continuous Integration và Continuous Deployment theo đúng yêu cầu bài.

Xem nội dung

Nhấn nút để hiển thị thời gian hiện tại.

1. CI/CD là gì?

CI - Continuous Integration

Tự động build và kiểm tra khi có commit/push mới để phát hiện lỗi sớm, giảm xung đột khi làm việc nhóm.

CD - Continuous Deployment

Sau khi pipeline thành công, phiên bản mới được triển khai tự động lên môi trường chạy thật, ở đây là GitLab Pages.

2. Luồng pipeline

Commit & Push
→
Build
→
Test
→
Deploy

Mỗi stage chỉ chạy khi stage trước đó thành công. Nếu test lỗi thì deploy sẽ không chạy.

3. Thành phần chính trong GitLab CI/CD

Pipeline

Tập hợp toàn bộ jobs được kích hoạt mỗi lần push code.

Stages

Các giai đoạn chạy tuần tự như build, test và deploy.

Jobs

Từng tác vụ cụ thể chứa script shell cần thực thi.

Artifacts

Kết quả đầu ra được chuyển tiếp giữa các stage, ví dụ thư mục public/.

4. Cấu trúc nộp bài

  • .gitlab-ci.yml đặt ở thư mục gốc repository
  • Thư mục public/ chứa index.html, styles.css, js/main.js
  • Pipeline chạy thành công trên branch mặc định
  • GitLab Pages hiển thị website live
  • Chụp minh chứng pipeline và live URL

5. Câu hỏi ôn tập

CI khác CD ở đâu?

CI tập trung build và test tự động; CD tiếp tục triển khai phiên bản đã pass lên môi trường chạy thật.

Runner là gì?

Runner là máy chủ hoặc container thực thi các jobs trong pipeline của GitLab.

Artifacts dùng để làm gì?

Artifacts lưu các file output của job để stage sau tiếp tục sử dụng.
